# Demo Prompts

Use these prompts to demonstrate the repo publicly.

## 1. End-to-End Assignment Execution

```text
Use $runpro on this project folder. It contains the assignment brief, draft, teacher comments, rubric screenshots, and source files. Reconstruct the real requirements, build the right execution path, and continue until the project is genuinely submission-ready or blocked by a real external constraint.
```

## 2. Conservative High-Score Audit

```text
Use $checkpro on this literature review. Reconstruct the task from the brief and rubric, audit the draft conservatively, and tell me whether it is actually 90+ plausible. Do not give a courtesy pass if the argument, evidence, or rubric coverage is still weak.
```

## 3. Format-Preserving Reinsertion

```text
Use $replacewords. I have an original DOCX and a revised text version. Replace the wording back into the original document while preserving the original formatting, font, paragraph settings, citation layout, and overall structure.
```

## 4. Minimal-Change Revision After Feedback

```text
Use $minimal-revision. This draft is already acceptable and I do not want a broad rewrite. Apply the supervisor comments with the smallest necessary changes while keeping the existing style and avoiding unnecessary rephrasing.
```

## 5. Paper-To-PPT Delivery

```text
Use $pptpro. Turn this paper/report into a visually strong editable PPTX. Reuse the original figures when they matter, avoid generic bullet slides, use conclusion-style slide titles where appropriate, and keep iterating until the deck looks intentional rather than template-like.
```

## 6. Final Submission Cleanup

```text
Use $final-delivery-clean on the final document package. Remove workflow notes, process-facing comments, workspace references, and any internal status language so the output reads like a clean deliverable.
```

## Best Public Demo Order

If you are posting a thread, GIF set, or short walkthrough, use this order:

1. `runpro` intake from a messy folder
2. `checkpro` catching score-relevant weaknesses
3. `replacewords` preserving the native document formatting
4. `pptpro` producing a real deck with figures
5. `final-delivery-clean` removing workflow residue

This shows a full delivery system, not just isolated prompts.
