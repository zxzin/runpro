# Launch Playbook

## Positioning

Lead with:

- submission-ready academic delivery
- source-grounded writing and auditing
- paper-to-PPT and report-to-PPT workflows
- format-preserving document revision
- bilingual English-plus-Chinese usefulness

Do not lead with:

- policy-evasion framing
- shortcut claims that imply replacing the student's work
- generic prompt engineering

That framing will reduce trust and make the repo easier to dismiss.

## Recommended Public Name

Best default:

`Run Pro`

Why:

- short
- easy to remember
- aligned with the orchestrator skill name, `runpro`
- broad enough to cover the companion academic delivery skills

## Recommended GitHub Description

> Run Pro Codex skill stack for project-folder execution, academic delivery, audit, revision, and paper-to-PPT workflows.

## Recommended README Hook

> Most prompt packs stop at drafting. This one is built for delivery closure: messy inputs in, submission-ready academic outputs out.

## Strongest Storyline

The repo is not "a bunch of prompts."

It is:

1. intake and scope reconstruction
2. source-grounded drafting
3. conservative academic audit
4. format-safe revision reinsertion
5. real presentation production
6. final deliverable cleanup

That is the story people remember and star.

## Best Skills To Feature First

Feature these on the repo front page and first public posts:

- `runpro`
- `checkpro`
- `replacewords`
- `pptpro`

Mention these as supporting utilities:

- `minimal-revision`
- `final-delivery-clean`

Keep this out of the main headline:

- policy-evasion or shortcut-submission framing

Those topics weaken the repo's credibility. Keep the public story focused on academic delivery quality, source grounding, format preservation, and final handoff.

## Best Launch Sequence

### Post 1: The Hook

Show that this is a full stack:

- folder of messy assignment materials
- Codex reconstructs scope
- audit catches real weaknesses
- final deck and final document come out clean

### Post 2: The Unique Trick

Show `replacewords`.

That is one of the most visually legible and unusual capabilities in the pack.

### Post 3: The Design Win

Show `pptpro` before/after results:

- generic deck
- repaired deck with real figures and stronger layout logic

### Post 4: The Quality Win

Show `checkpro` catching a false `90+ plausible` judgement or identifying why a draft is only complete, not genuinely high-scoring.

## What Will Most Likely Get Stars

- a crisp README
- one compelling GIF or screenshot sequence for `replacewords`
- one strong before/after for `pptpro`
- one concrete rubric-audit example for `checkpro`
- clear English-facing framing

## What Will Most Likely Kill Momentum

- marketing it mainly as anti-AI detection tooling
- uploading a private messy skill dump without repo story
- burying the strongest skills under too many unrelated ones
- making the repo feel China-only when it is actually bilingual
